🟣 T. Potencia 🩵 T. Energía 🟡 Excesos 🔴 Imp. Eléctrico ⬜ Alquiler 🟠 IVA 🟢 Total
// ── Step 1: Power term ────────────────────────────────────────────── terminoPotencia = Σ ( precioP_Pi × kW_Pi × dias / 365 ) // for each active power period // ── Step 2: Energy term ───────────────────────────────────────────── terminoEnergia = Σ ( precioE_Pi × kWh_Pi ) // for each active energy period // ── Step 3: Base imponible (IE base) ──────────────────────────────── baseImponible = terminoEnergia + terminoPotencia + excesos + reactiva + otrosCargos // ── Step 4: Impuesto Eléctrico ────────────────────────────────────── impuestoElectrico = baseImponible × 0.0511269 // 5.11269% — fixed by law // ── Step 5: IVA base (alquiler added here, not in IE base) ────────── baseIva = baseImponible + impuestoElectrico + alquilerMedida // ── Step 6: IVA ───────────────────────────────────────────────────── iva = baseIva × 0.21 // 21% (or override) // ── Step 7: Total ─────────────────────────────────────────────────── total = baseIva + iva // ── Step 8: Savings vs current bill ───────────────────────────────── ahorro = facturaActual - total pctAhorro = ( ahorro / facturaActual ) × 100

📗 Excel (COMPARATIVA LUZ, col Q)

Q40 = Σ(kW_Pi × I43_Pi) × Q23 ← T.POTENCIA Q41 = Σ(kWh_Pi × I48_Pi) ← T.ENERGIA Q42 = Q24 ← EXCESOS Q43 = (Q40+Q41+Q42+Q44) × E51 ← IMPUESTOS (IE) Q44 = Q26 ← OTROS CARGOS Q45 = E48 ← ALQUILER Q46 = (Q40+Q41+Q42+Q43+Q44+Q45)×E50 ← IVA Q49 = Q40+Q41+Q42+Q43+Q44+Q45+Q46 ← TOTAL

💻 App (calculationService.ts)

terminoPotencia = Σ price×kW×(dias/365) terminoEnergia = Σ price×kWh exceso = excesoPotencia baseImponible = ener+pot+exce+otros impElec = baseImponible × 0.0511269 baseIva = baseImponible+IE+alquiler iva = baseIva × 0.21 total = baseIva + iva
💡 Key insight: alquiler is outside the IE base Meter rental (alquiler equipo medida) is NOT included in the Impuesto Eléctrico calculation base — it goes directly into the IVA base. This matches the Excel formula exactly (Q45 is added to Q46 but not Q43).

🏷️ 5. Product Types & How Energy Price Is Determined

Fixed

FIJO Products

Estable, Estable Plus, 1P Plus, 1P Plus XL, Estable Talleres, Estable Plus Talleres

precioE = DB lookup // key: ELEC:FIJO:{PRODUCT}: // {TIER}:{TARIFF}:{P}:ENERGIA

Price is fixed per period — doesn't change with OMIE market.

Indexed

INDEX Products

Dinámica, Dinámica Plus, Dinámica Control, Dinámica Control Plus, Dinámica Control Techo

precioE = OMIE_Pi + margen // OMIE: user-entered estimate // margen: from DB lookup // key: ELEC:INDEX:{PRODUCT}: // {TIER}:{TARIFF}:{P}:MARGEN

Energy price tracks the wholesale OMIE market + a fixed margin.

Custom

Personalizada Products

Personalizada Index & Personalizada OMIE + B

// Personalizada Index: precioE = OMIE_Pi + margenEnergia_Pi/1000 // Personalizada OMIE+B: precioE = OMIE_Pi + terminoB_Pi/1000 // margins entered per-period by agent

Agent manually enters per-period margins. No DB lookup needed for energy.

💰 7. Savings Calculation

How ahorro is computed

ahorro = facturaActual - total pctAhorro = (ahorro / facturaActual) × 100 ahorroAnual = ahorro × 12
⚠️ Annual savings ahorroAnual is simply ahorro × 12 — it assumes all months are identical. For full-year simulations (365 days) the Excel uses the actual 12-month sum.

Excel equivalent

// Q10 = AHORRO ANUAL IF(E25=365, Q32-Q49, Q16×12) // E25 = dias; Q32 = competencia total // Q49 = Axpo total; Q16 = monthly saving // Q15 = % saving Q15 = Q16 / Q32

🛠️ 8. Personalizada Products — Deep Dive

These are fully customisable products where the commercial agent enters per-period margins manually. The Excel has dedicated sheets PERSONALIZADA INDEX and PERSONALIZADA OMIE + B.

Personalizada Index

Agent sets a margen energía (€/MWh) and margen potencia per period.

// Energy cost per period: e_Pi = (OMIE_Pi + margenE_Pi/1000) × kWh_Pi // Power cost per period: p_Pi = (margenPot_Pi/1000) × kW_Pi × dias/365

Personalizada OMIE + B

Agent sets a término B (€/MWh) and margen potencia per period.

// Energy cost per period: e_Pi = (OMIE_Pi + terminoB_Pi/1000) × kWh_Pi // Power cost per period: p_Pi = (margenPot_Pi/1000) × kW_Pi × dias/365
📐 /1000 conversion Margins are entered in €/MWh (as shown in the Excel sheets) but all calculation is done in €/kWh, so they are divided by 1000.
